# 产品需求文档 (PRD) - 旧物志 (Relic Archive)

## 1. 产品概述

### 1.1 产品名称
- **暂定名称**：旧物志 / Relic / Archive

### 1.2 核心定位
为恋物癖及收藏型用户打造的“旧物数字档案馆”，提供一个「情绪型数字收藏空间」。
- **核心价值**：记录、分类、情绪赋值、数字重构、私密收藏。
- **非核心定位**：非二手交易平台、非断舍离工具。

### 1.3 目标用户
- **收藏癖用户**：希望完整保存物品细节。
- **恋物者/物品依恋者**：重视物品的磨损痕迹和情感连接。
- **怀旧者**：希望通过数字方式留住物品的记忆。
- **强迫型收藏者**：追求分类的准确性和整理的秩序感。

---

## 2. 核心功能模块

### 2.1 模块一：多角度拍照记录系统
**目标**：完整记录旧物的“存在感”和“细节感”。

- **功能需求**：
    - 多角度拍摄引导（自动提示补拍缺失角度）。
    - 局部细节拍摄（支持微距拍摄磨损、折痕、污渍、标签）。
    - 视频短拍（可选）。
    - 时间轴记录（拥有时间、重要事件节点）。
    - 拍摄备注（支持文字 + 语音情感记录）。
- **AI 辅助**：
    - 自动识别物体边界。
    - 自动光线校正。
    - 自动去背景（可选）。

### 2.2 模块二：自动分类系统
**目标**：智能整理，降低用户收纳成本，同时满足强迫症式的精细管理。

- **一级分类建议**：
    - 衣物、电子用品、家具、玩具、纸制品、生活用品、私密用品（支持隐藏）、自定义分类。
- **AI 分类逻辑**：
    - 基于图像识别和物体特征。
    - 学习用户历史分类习惯。
- **标签系统**：
    - **物理属性**：年代、材质、保存状态。
    - **使用属性**：使用频率。
    - **情绪标签（核心）**：安全感、怀旧、私密、欲望、放不下、纪念。

### 2.3 模块三：AI 重构系统（核心差异点）
**目标**：赋予静态物品以“故事感”和“生命力”。

- **自动生成旧物描述**：
    - **生成维度**：外观描述、使用痕迹、推测年代、材质分析、情绪氛围。
    - **风格选择**：客观档案式、温柔叙述式、博物馆标签式、私密日记式。
- **3D 重构（MVP 阶段简化）**：
    - **MVP 方案**：多图拼接、视差动效、轻微旋转动画（模拟 3D）。
    - **未来规划**：基于 Gaussian Splatting 的高质量 3D 模型、AR 投影、复原模式（展示全新状态）、时间侵蚀模拟。

### 2.4 模块四：收藏空间系统
**目标**：提供沉浸式的浏览体验。

- **功能**：
    - 创建收藏馆（设定主题）。
    - 隐私设置（公开/私密）。
    - 虚拟展览生成（瀑布流、网格、沉浸式）。

### 2.5 模块五：隐私与心理安全机制
**目标**：构建无评判、高安全性的私密空间。

- **功能**：
    - 本地加密存储。
    - FaceID/生物识别解锁。
    - 伪装入口（可选）。
    - 强隐私模式（默认不分享）。

## 3. 用户故事 (User Stories)

| 角色 | 需求 | 目的/价值 |
| :--- | :--- | :--- |
| **收藏癖用户** | 多角度拍摄旧物 | 完整保存细节，不错过任何角落。 |
| **恋物者** | 记录磨损痕迹 | 因为痕迹是物品最重要的部分，承载了记忆。 |
| **物品依恋者** | 系统自动分类 | 减少整理负担，专注于情感体验。 |
| **强迫型收藏者** | 手动微调分类 | 确保分类的绝对准确，获得心理安宁。 |
| **情绪依附型用户** | AI生成“故事感描述” | 感觉物品被理解，赋予物品“灵性”。 |
| **私密收藏者** | 所有的收藏都不被看见 | 保护隐私，避免社交压力和评判。 |

## 4. 用户行为路径 (User Flow)

### 4.1 核心路径 1：第一次记录旧物
1.  **启动**：打开 APP -> 首页 -> 点击“开始记录”。
2.  **拍摄**：进入拍摄引导 -> 多角度拍摄/细节特写 -> 确认。
3.  **处理**：AI 自动识别分类 -> AI 生成描述（用户可编辑/重选风格）。
4.  **归档**：生成伪 3D/预览 -> 进入物品档案页 -> 加入收藏馆。

### 4.2 核心路径 2：浏览收藏
1.  **进入**：打开 APP -> “我的收藏馆”。
2.  **浏览**：瀑布流/网格展示 -> 点击特定旧物。
3.  **互动**：查看 3D/动态效果 -> 阅读 AI 故事 -> 查看时间轴。

### 4.3 核心路径 3：深度情绪使用（夜间模式）
1.  **场景**：夜间模式打开 -> 进入“故事模式”。
2.  **体验**：播放背景白噪音/音乐 -> 360° 旋转查看（或视差浏览） -> 沉浸式回忆。

## 5. UI 设计方向

### 5.1 视觉风格
- **方向一：私密档案馆**（推荐）
    - 关键词：暗色背景、柔和暖光、低对比度、玻璃拟态、沉浸感。
- **方向二：博物馆数字化**
    - 关键词：极简、大留白、细线条、档案编号、冷色调。
- **方向三：私人日记**
    - 关键词：手写体、纸张纹理、暖黄背景、微光粒子。

### 5.2 关键交互
- **拍摄引导**：半透明轮廓提示、进度环形指示。
- **浏览体验**：黑色背景、单光源打光、缓慢缩放动画。
- **标签设计**：低饱和色、圆角胶囊样式、可拖动。

## 6. 商业模式与扩展

### 6.1 商业模式 (Freemium)
- **免费版**：基础拍照、基础分类、基础 AI 描述、数量限制（如 50 件）。
- **Pro 版**：无限存储、高清重构、多风格故事、加密空间、云端备份。
- **增值服务**：AI 复原模式、深度情绪报告、定制虚拟展馆。

### 6.2 风险提示
- **心理健康**：避免诱导过度依赖，强调“记录”而非“沉迷”。
- **内容合规**：严格过滤违规内容（尤其是私密上传）。
- **成本控制**：3D 模型和高清图片存储成本较高，需优化压缩算法。

### 6.3 扩展方向
- 数字遗产传承。
- AR 旧物房间。
- 物品情绪分析报告。
