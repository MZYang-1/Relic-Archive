# MVP 开发计划与里程碑

## 1. MVP 目标
**核心验证**：用户是否愿意持续记录物品，并对 AI 生成的情绪描述产生共鸣。
**关键指标**：7日留存 > 30%，平均记录物品 ≥ 5 件。

## 2. 阶段划分

### 阶段一：基础架构与原型 (Week 1-2)
- **目标**：跑通“拍照 -> 上传 -> 存储”流程。
- **后端**：
    - 搭建 API 网关、用户认证系统。
    - 数据库设计与初始化。
    - 文件上传接口 (S3/OSS)。
- **移动端**：
    - 项目初始化 (React Native/Flutter 或原生)。
    - 基础 UI 框架搭建。
    - 相机模块集成（基础拍照功能）。

### 阶段二：核心功能开发 - 记录与 AI (Week 3-4)
- **目标**：实现“智能记录”体验。
- **AI 服务**：
    - 集成图像识别 API (分类)。
    - 对接 LLM API (GPT-4o/Claude)，调试 Prompt 以生成高质量“情绪文案”。
- **移动端**：
    - 多角度拍摄引导 UI。
    - 物品详情页开发（展示图片、AI 描述）。
    - 编辑功能（修改分类、标签、描述）。

### 阶段三：收藏馆与伪 3D 体验 (Week 5-6)
- **目标**：提升浏览体验，实现 MVP 级别的“重构”。
- **移动端**：
    - 实现“伪 3D”效果（如通过滑动切换多角度照片，或简单的视差动画）。
    - 收藏馆页面开发（瀑布流、分类筛选）。
    - 夜间模式/沉浸模式 UI 适配。

### 阶段四：隐私、优化与发布 (Week 7-8)
- **目标**：确保安全，打磨细节，准备上线。
- **功能**：
    - 本地加密、FaceID/指纹解锁集成。
    - 私密空间逻辑实现。
- **优化**：
    - 图片加载速度优化。
    - 动画流畅度调整。
    - Bug 修复。
- **发布**：
    - 内测邀请。
    - App Store / Google Play 上架准备。

## 3. MVP 功能裁剪表

| 功能模块 | MVP 包含 | MVP 暂缓/替代方案 |
| :--- | :--- | :--- |
| **3D 重构** | 否 | **替代**：多图滑动查看、视差动效。 |
| **AR 投影** | 否 | 后续版本迭代。 |
| **社交分享** | 否 | 仅支持生成图片分享，不做站内社交。 |
| **视频记录** | 是 (基础) | 限制时长，暂不提供复杂剪辑。 |
| **AI 自动去背** | 否 | 仅提供原图展示，去背成本高且非必须。 |
| **多语言支持** | 否 | 仅支持单语言（如中文）。 |

## 4. 资源需求

- **人力**：
    - 1 后端/AI 工程师
    - 1 移动端工程师
    - 1 UI/UX 设计师
- **基础设施**：
    - 云服务器 (AWS/Aliyun)
    - 对象存储服务
    - LLM API Token (OpenAI/Anthropic)
